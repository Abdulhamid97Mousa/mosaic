### Official Gymnasium Documentation Reference

(See references/FROZEN_LAKE_GYMNASIUM_DOCS.md for full details.)

- **Tile letters:**
  - `S` = Start tile (agent initial position, default (0,0))
  - `G` = Goal tile (default (3,3) for 4x4, (7,7) for 8x8)
  - `F` = Frozen tile
  - `H` = Hole tile

- **Default maps:**
  - 4x4: `['SFFF', 'FHFH', 'FFFH', 'HFFG']`
  - 8x8: `['SFFFFFFF', 'FFFFFFFF', 'FFFHFFFF', 'FFFFFHFF', 'FFFHFFFF', 'FHHFFFHF', 'FHFFHFHF', 'FFFHFFFG']`

- **Arguments:**
  - `desc`: Custom map (list of strings)
  - `map_name`: "4x4" or "8x8" (uses default map if desc is None)
  - `is_slippery`: True/False
  - `success_rate`: Probability of intended direction (default 1/3)
  - `reward_schedule`: (Goal, Hole, Frozen) rewards (default (1,0,0))

- **Random hole placement:**
  - If both `desc` and `map_name` are None, a random 8x8 map is generated (80% frozen tiles).

### GUI_BDI_RL Integration

- **Configurable parameters in UI:**
  - Grid size (height, width)
  - Start position (dropdown, default (0,0))
  - Goal position (dropdown, default (7,7) for 8x8)
  - Hole count
  - Random hole placement (checkbox, default False)
  - Slippery ice, success rate, reward schedule

- **Behavior:**
  - When random holes is unchecked and grid is 4x4 or 8x8 with default start/goal, the official Gymnasium map is used.
  - When random holes is checked, holes are placed randomly (count as specified).

- **Adapter logic:**
  - Generates map descriptor based on config and random_holes flag.
  - Passes correct desc/map_name to Gymnasium API.
